Contexte et objectif

Dans ce projet, j'ai configuré un environnement de développement utilisant VirtualBox pour exécuter deux systèmes d'exploitation différents, Linux et Windows, sur une même machine physique. L'objectif était de permettre la communication entre les deux systèmes en utilisant Samba pour le partage de fichiers et SSH pour l'accès distant sécurisé à la machine Linux.

Ce projet m'a permis de découvrir la virtualisation, les principes de mise en réseau entre systèmes et les méthodes d'administration de serveurs distants.

Étapes de configuration

Voici les principales étapes que j'ai suivies pour réaliser ce projet :

  1. Installation de VirtualBox : J'ai installé VirtualBox, un logiciel de virtualisation, sur mon poste de travail. Ce logiciel me permet de créer et gérer des machines virtuelles pour exécuter différents systèmes d'exploitation en parallèle.
  2. Installation de Linux et Windows : J'ai créé deux machines virtuelles. La première exécute Ubuntu Linux et la seconde Windows 10. Chaque système est installé sur une machine virtuelle distincte avec les ressources nécessaires (mémoire, processeur, etc.).
  3. Configuration réseau : Pour que les machines virtuelles puissent communiquer entre elles, j'ai configuré le réseau de VirtualBox en mode réseau interne, ce qui permet aux deux machines d'être sur le même réseau local privé.
  4. Installation de Samba sur Linux : J'ai installé Samba sur la machine Linux afin de permettre le partage de fichiers avec Windows. Cela permet à la machine Windows d'accéder à des dossiers spécifiques sur la machine Linux.
  5. Configuration de Samba : Après avoir installé Samba, j'ai configuré les fichiers de partage (/etc/samba/smb.conf) pour définir les répertoires partagés et permettre l'accès depuis la machine Windows.
  6. Test de la communication Samba : J'ai vérifié que le partage de fichiers fonctionne en accédant aux répertoires partagés depuis Windows via l'explorateur de fichiers. La machine Windows peut maintenant lire et écrire dans les dossiers partagés de Linux.
  7. Installation et configuration de SSH sur Linux : J'ai installé le serveur OpenSSH sur la machine Linux pour permettre une connexion sécurisée via le protocole SSH depuis Windows ou d'autres machines.
  8. Connexion SSH depuis Windows : J'ai utilisé un client SSH sur Windows (comme PuTTY) pour me connecter à la machine Linux à distance, ce qui me permet d'exécuter des commandes à partir de la ligne de commande de Windows tout en contrôlant la machine Linux.

Problèmes rencontrés et solutions

Tout au long de ce projet, j'ai rencontré plusieurs problèmes que j'ai dû résoudre :

  • Problème de réseau entre les machines virtuelles : Au départ, les machines virtuelles ne parvenaient pas à se connecter entre elles. J'ai résolu ce problème en vérifiant les paramètres de réseau dans VirtualBox et en m'assurant qu'elles étaient bien sur le même réseau interne.
  • Problème de permission Samba : Lors de l'accès aux fichiers partagés sur Linux depuis Windows, un problème de permission est apparu. J'ai résolu cela en ajustant les paramètres de permissions dans les fichiers de configuration de Samba pour autoriser l'accès en lecture et en écriture aux utilisateurs de Windows.
  • Problème d'authentification SSH : J'ai eu un problème de connexion SSH dû à une mauvaise configuration de l'utilisateur Linux. J'ai corrigé cela en vérifiant les fichiers de configuration d'OpenSSH et en m'assurant que l'utilisateur avait bien les permissions nécessaires pour se connecter via SSH.

Tests et validation

Une fois toutes les configurations terminées, j'ai effectué des tests pour m'assurer que tout fonctionnait correctement :

  • Test Samba : J'ai vérifié que les répertoires partagés sur Linux étaient accessibles depuis Windows. Cela a permis de tester la connexion réseau et le partage de fichiers.
  • Test SSH : J'ai testé la connexion SSH en accédant à la machine Linux depuis Windows via PuTTY, et cela a fonctionné comme prévu. J'ai pu exécuter des commandes Linux à distance et manipuler des fichiers.
  • Test de la communication réseau : J'ai vérifié que les machines pouvaient se pinger entre elles en utilisant les adresses IP locales assignées dans VirtualBox. La communication a été réussie.

Ce que j'ai aimé

- L'installation et la configuration d'un environnement complet pour le développement sur deux systèmes d'exploitation différents.

- L'apprentissage de la gestion de serveur et de partage de fichiers via Samba, ainsi que l'utilisation de SSH pour contrôler les machines à distance.

- La résolution de problèmes techniques qui m'ont permis d'améliorer mes compétences en gestion de systèmes.

Ce que j'ai pas aimé

- Le fait que l'installation initiale de certains outils (comme Samba) ait pris plus de temps que prévu.

- Les petites difficultés de communication entre Linux et Windows, notamment au début, qui ont nécessité pas mal de dépannage.

Conclusion

Ce projet m'a permis de comprendre la virtualisation et la configuration de machines virtuelles sur un même poste de travail. J'ai également acquis des compétences en gestion des réseaux entre machines, en partage de fichiers via Samba, et en administration système via SSH. Cette expérience m'a donné une meilleure compréhension de la manière dont différents systèmes peuvent être configurés pour travailler ensemble dans un environnement de développement.
